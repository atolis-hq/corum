delivery overlay

milestones / epics / stories / tickets

we need a way to annotate designs with a delivery plan.
Initially we would probably design a whole feature - is that a branch or multiple on one branch?
once the design is done, we need to be able to slice it into stories. the agent might work this out, or there could be some tool to find clusters of deliverables. 

Even at field level, we might annotate what is needed for a story.

This applies to everything, some bulk tools might be needed to mark things up. it needs some thought about a clean way to make this work. Also how the data is stored and how the nodes are extended.

stage 2: A story / ticket / task could be created in the tool and then synced to jira or some other tool, storing the external ticket reference. alternatively tickets could be imported from jira.
