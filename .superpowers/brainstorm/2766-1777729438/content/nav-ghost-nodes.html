When viewing an overlay, what appears in the nav tree?

When you select a branch to overlay (e.g. feat/checkout-v2 over main), the nav tree on the left could behave in different ways. Which matches your intent?

A

Nav shows viewing branch only

The nav tree only lists nodes that exist on the currently viewed branch (e.g. main). Ghost nodes from other branches are only visible inside the node detail view — you'd navigate to a node on main and see ghosted incoming fields alongside it.

nav tree
▸ orders
◉ Order (main)
◉ OrderLine (main)
B

Nav shows ghost nodes too

Ghost nodes from incoming branches appear in the nav tree with a muted/ghosted style. Clicking one opens a view of that node in its originating branch context, with a badge showing it doesn't exist on the viewed branch.

nav tree
▸ orders
◉ Order
◉ OrderLine
◎ CheckoutSession ghost
C

Nav shows all, with signal dots

All nodes appear in the nav. Nodes that have overlay activity (incoming changes, ghost presence, conflicts) get a small signal dot. Click any node to see its full overlay view. No ghosting in the nav itself — just a presence indicator.

◉ Order
◉ OrderLine
◎ CheckoutSession
